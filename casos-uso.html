Interacciones de los actores con el sistema RMR Fashion

Diagrama de Casos de Uso

Descripción de Casos de Uso

ID Caso de Uso Actor(es) Descripción Precondición
CU01 Registrarse Cliente El cliente crea una cuenta en el sistema proporcionando nombre, correo y contraseña. El cliente no tiene cuenta registrada previamente.
CU02 Iniciar sesión Cliente Admin El usuario (cliente o admin) ingresa sus credenciales para acceder al sistema. El usuario tiene una cuenta activa registrada.
CU03 Ver catálogo Cliente El cliente navega por los productos disponibles visualizando imágenes, precios, tallas y colores. El cliente ha accedido al sitio web.
CU04 Agregar al carrito Cliente El cliente selecciona un producto y especifica cantidad/talla para añadirlo al carrito de compras. El cliente está autenticado y hay productos en el catálogo.
CU05 Realizar pedido Cliente El cliente confirma los productos en el carrito, ingresa datos de envío y finaliza la compra. El carrito contiene al menos un producto y el cliente está autenticado.
CU06 Realizar pago Sistema de pagos El sistema procesa el pago del pedido a través de pasarela externa (tarjeta, transferencia, etc.). El cliente ha confirmado el pedido y proporcionado datos de pago válidos.
CU07 Gestionar productos Administrador El administrador agrega, edita, activa/desactiva o elimina productos del catálogo. El administrador ha iniciado sesión con rol de admin.
CU08 Gestionar usuarios Administrador El administrador puede visualizar, activar, suspender o eliminar cuentas de clientes. El administrador ha iniciado sesión con rol de admin.
CU09 Ver historial de pedidos Cliente El cliente consulta sus compras anteriores con detalles de estado, fecha y total. El cliente está autenticado y ha realizado al menos un pedido previo.
CU10 Actualizar perfil Cliente El cliente modifica sus datos personales, dirección o contraseña. El cliente está autenticado en el sistema.
